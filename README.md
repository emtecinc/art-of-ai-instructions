# art-of-ai-instructions

Centralized instruction intelligence for AI-assisted Playwright test automation and manual test generation.

> This repo ships **agents**, **instructions**, and **utilities**.
> Downstream projects clone this repo and copy assets into `.github/agents/`, `.github/instructions/`, and `.github/utilities/`.

## What This Is

A shared repository providing centralized agents, reusable framework rules, coding patterns, utility guidance, and AI-readable discovery for Salesforce Playwright test automation. Consuming projects clone this repo and copy assets into `.github/` to ensure consistent, high-quality AI-generated code across all project instances.

## Architecture: Centralized vs Local

### Centralized (this repo вҖ” copied into `.github/` in consuming projects)

| What | Consumed Under | Why |
|---|---|---|
| `agents/*.agent.md` | `.github/agents/` | Centralized agent definitions вҖ” copied to each consuming project |
| `instructions/*.instructions.md` | `.github/instructions/` | Framework rules auto-loaded via `applyTo` вҖ” single authority per concern |
| `utilities/*.md` | `.github/utilities/` | Helper utility reference docs вҖ” read on demand |
| `README.md`, `INDEX.md`, `AGENTS.md` | `.github/` | Discovery, indexing, agent registry |

### Local (consuming project)

| What | Why |
|---|---|
| `.github/copilot-instructions.md` | Project-specific context (org URL, app names, object list) |
| `.github/local-override.md` | Project-level rule overrides (use sparingly) |
| Source code: `pages/`, `workflows/`, `tests/`, `test-data/` | Generated by agents |
| `.env` | Credentials and environment variables |

### Instruction Loading Flow

1. Agent edits a file under `pages/` вҶ’ `.github/instructions/page-objects.instructions.md` auto-loads via `applyTo: "pages/**/*.ts"`
2. Agent edits under `workflows/` вҶ’ `.github/instructions/workflows.instructions.md` auto-loads
3. Agent edits under `tests/` вҶ’ `.github/instructions/spec-files.instructions.md` + `.github/instructions/salesforce-stability.instructions.md` auto-load
4. Agent edits under `test-data/` вҶ’ `.github/instructions/test-data.instructions.md` auto-loads

### Instruction Precedence

| Priority | Source | Scope |
|---|---|---|
| 1 (highest) | `.github/instructions/*.instructions.md` | Reusable framework rules вҖ” always active, auto-loaded |
| 2 | `.github/copilot-instructions.md` | Project-specific context (org URL, objects, apps) |
| 3 | `.github/local-override.md` | Project-level exceptions вҖ” only overrides the specific behavior it explicitly mentions |

**Rule:** Centralized instructions are the default authority. Local files add context or override specific behaviors вҖ” they never replace the full instruction set.

## 3-Layer Test Architecture

All Playwright agents enforce a strict separation of concerns:

| Layer | Location | Responsibility |
|---|---|---|
| **Page Object** | `pages/<object>/` | Locators, element interactions, `expect()` assertions |
| **Workflow** | `workflows/<object>/` | Business-level step orchestration via `this.step()` |
| **Spec** | `tests/<object>/` | Declarative test orchestration вҖ” zero DOM, zero assertions |

## Agent Pipeline

```
Planner в”Җв”Җ(plan)в”Җв”Җв–ә Generator в”Җв”Җ(code)в”Җв”Җв–ә Healer
```

| Agent | Purpose | Location |
|---|---|---|
| **Planner** | Explores live app, designs test scenarios, saves plans to `specs/` | `agents/` |
| **Generator** | Implements plans as Playwright code following 3-layer architecture | `agents/` |
| **Healer** | Debugs and fixes failing tests without violating framework rules | `agents/` |
| **Manual Test Step Generator** | Converts Playwright codegen output to structured manual test cases | `agents/` |

See [AGENTS.md](AGENTS.md) for detailed agent boundaries and dependencies.

## How to Set Up a Consuming Project

### 1. Clone and copy assets

Clone this repo and copy the required folders into your project's `.github/` directory:

```bash
# Copy agents, instructions, and utilities into .github/
cp -r <centralized-repo>/agents/ .github/agents/
cp -r <centralized-repo>/instructions/ .github/instructions/
cp -r <centralized-repo>/utilities/ .github/utilities/
```

This makes all centralized files accessible at `.github/agents/`, `.github/instructions/`, `.github/utilities/`.

### 2. Create local files

In `.github/`:
- **`copilot-instructions.md`** вҖ” project context (org URL, app names, object list, custom behaviors)
- **`local-override.md`** вҖ” rule overrides (use sparingly вҖ” most rules should stay centralized)

### 3. Use agents

In GitHub Copilot Chat:
- `@playwright-test-planner` вҖ” create test plans
- `@playwright-test-generator` вҖ” generate test code from plans
- `@playwright-test-healer` вҖ” debug failing tests
- `@AI-test-case-step-generator` вҖ” produce manual test cases from codegen output
