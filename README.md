# art-of-ai-instructions

Centralized instruction intelligence for AI-assisted Playwright test automation and manual test generation.

> **This repo is instructions-only.** It contains framework rules, utility reference docs, and discovery files.
> It does NOT contain or ship any agent files. ALL agents live in consuming projects under `.github/agents/`.

## What This Is

A shared instruction repository providing reusable framework rules, coding patterns, utility guidance, and AI-readable discovery for Salesforce Playwright test automation. Consuming projects reference this repo (under `.github/ai-instructions/`) to ensure consistent, high-quality AI-generated code across all project instances.

## Architecture: Centralized vs Local

### Centralized (this repo — consumed under `.github/ai-instructions/` in consuming projects)

| What | Consumed Under | Why |
|---|---|---|
| `instructions/*.instructions.md` | `.github/ai-instructions/instructions/` | Framework rules auto-loaded via `applyTo` — single authority per concern |
| `utility/*.md` | `.github/ai-instructions/utility/` | Helper utility reference docs — read on demand |
| `README.md`, `INDEX.md`, `AGENTS.md` | `.github/ai-instructions/` | Discovery, indexing, agent registry |

### Local (consuming project)

| What | Why |
|---|---|
| `.github/copilot-instructions.md` | Project-specific context (org URL, app names, object list) |
| `.github/local-override.md` | Project-level rule overrides (use sparingly) |
| `.github/agents/playwright-test-planner.agent.md` | Requires local MCP server + live browser |
| `.github/agents/playwright-test-generator.agent.md` | Requires local MCP server + live browser |
| `.github/agents/playwright-test-healer.agent.md` | Requires local MCP server + live browser |
| `.github/agents/manual-test-step-generator.agent.md` | Standalone text-only agent — maintained locally |
| Source code: `pages/`, `workflows/`, `tests/`, `test-data/` | Generated by agents |
| `.env` | Credentials and environment variables |

### Why ALL Agents Are Local

All agents — including the Manual Test Step Generator — live in `.github/agents/` in each consuming project because:
- **Playwright agents** depend on MCP tool server, live browser, and local workspace
- **All agents** benefit from local ownership for versioning, customization, and tool configuration
- This repo provides the **instruction intelligence** that agents follow, not the agents themselves

### Instruction Loading Flow

1. Agent edits a file under `pages/` → `.github/ai-instructions/instructions/page-objects.instructions.md` auto-loads via `applyTo: "pages/**/*.ts"`
2. Agent edits under `workflows/` → `.github/ai-instructions/instructions/workflows.instructions.md` auto-loads
3. Agent edits under `tests/` → `.github/ai-instructions/instructions/spec-files.instructions.md` + `.github/ai-instructions/instructions/salesforce-stability.instructions.md` auto-load
4. Agent edits under `test-data/` → `.github/ai-instructions/instructions/test-data.instructions.md` auto-loads

### Instruction Precedence

| Priority | Source | Scope |
|---|---|---|
| 1 (highest) | `.github/ai-instructions/instructions/*.instructions.md` | Reusable framework rules — always active, auto-loaded |
| 2 | `.github/copilot-instructions.md` | Project-specific context (org URL, objects, apps) |
| 3 | `.github/local-override.md` | Project-level exceptions — only overrides the specific behavior it explicitly mentions |

**Rule:** Centralized instructions are the default authority. Local files add context or override specific behaviors — they never replace the full instruction set.

## 3-Layer Test Architecture

All Playwright agents enforce a strict separation of concerns:

| Layer | Location | Responsibility |
|---|---|---|
| **Page Object** | `pages/<object>/` | Locators, element interactions, `expect()` assertions |
| **Workflow** | `workflows/<object>/` | Business-level step orchestration via `this.step()` |
| **Spec** | `tests/<object>/` | Declarative test orchestration — zero DOM, zero assertions |

## Agent Pipeline

```
Planner ──(plan)──► Generator ──(code)──► Healer
```

| Agent | Purpose | Location |
|---|---|---|
| **Planner** | Explores live app, designs test scenarios, saves plans to `specs/` | Local |
| **Generator** | Implements plans as Playwright code following 3-layer architecture | Local |
| **Healer** | Debugs and fixes failing tests without violating framework rules | Local |
| **Manual Test Step Generator** | Converts Playwright codegen output to structured manual test cases | Local |

See [AGENTS.md](AGENTS.md) for detailed agent boundaries and dependencies.

## How to Set Up a Consuming Project

### 1. Reference this repo

Add as a submodule or symlink so `instructions/` and `utility/` are accessible under `.github/ai-instructions/`:

```bash
git submodule add <repo-url> .github/ai-instructions
```

This makes all centralized files accessible at `.github/ai-instructions/instructions/`, `.github/ai-instructions/utility/`, etc.

### 2. Create local files

In `.github/`:
- **`copilot-instructions.md`** — project context (org URL, app names, object list, custom behaviors)
- **`local-override.md`** — rule overrides (use sparingly — most rules should stay centralized)

In `.github/agents/`:
- **`playwright-test-planner.agent.md`**
- **`playwright-test-generator.agent.md`**
- **`playwright-test-healer.agent.md`**
- **`manual-test-step-generator.agent.md`**

### 3. Use agents

In GitHub Copilot Chat:
- `@playwright-test-planner` — create test plans
- `@playwright-test-generator` — generate test code from plans
- `@playwright-test-healer` — debug failing tests
- `@AI-test-case-step-generator` — produce manual test cases from codegen output
